# Scroll Top 버튼

## 364줄

```css
.scroll-top {
```

`.scroll-top` class를 가진 요소를 선택합니다.

HTML에서는 이 버튼이었습니다.

```html
<button class="scroll-top" type="button" aria-label="맨 위로 이동" data-scroll-top>Top</button>
```

이 버튼은 페이지를 아래로 내렸을 때 나타나서, 누르면 맨 위로 이동하게 해주는 버튼입니다.

---

## 365줄

```css
position: fixed;
```

버튼을 화면 기준으로 고정합니다.

`fixed`는 문서의 위치가 아니라 브라우저 화면을 기준으로 배치합니다.

그래서 사용자가 페이지를 아래로 스크롤해도 버튼은 화면의 같은 위치에 계속 남아 있습니다.

---

## 366줄

```css
right: 1rem;
```

화면 오른쪽에서 `1rem`만큼 떨어뜨립니다.

즉, 버튼을 오른쪽 아래에 두기 위한 가로 위치 설정입니다.

---

## 367줄

```css
bottom: 1rem;
```

화면 아래쪽에서 `1rem`만큼 떨어뜨립니다.

366줄과 합치면:

```txt
오른쪽에서 1rem
아래쪽에서 1rem
```

이므로 버튼은 화면 오른쪽 아래에 고정됩니다.

---

## 368줄

```css
width: 52px;
```

버튼의 너비를 52px로 정합니다.

---

## 369줄

```css
height: 52px;
```

버튼의 높이를 52px로 정합니다.

너비와 높이가 같으므로 정사각형입니다.

```txt
52px × 52px
```

---

## 370줄

```css
opacity: 0;
```

처음에는 버튼을 투명하게 만듭니다.

`opacity`는 투명도입니다.

```txt
opacity: 1 → 완전히 보임
opacity: 0 → 완전히 안 보임
```

즉, 페이지 처음에는 `Top` 버튼이 보이지 않습니다.

---

## 371줄

```css
pointer-events: none;
```

마우스 클릭 같은 포인터 이벤트를 받지 않게 합니다.

370줄에서 이미 투명하게 만들었지만, 투명하다고 해서 클릭이 불가능한 것은 아닙니다.

그래서 이 줄을 같이 씁니다.

```txt
opacity: 0
→ 눈에는 안 보임

pointer-events: none
→ 클릭도 안 됨
```

이 두 줄이 같이 있어야 “진짜로 숨겨진 버튼”처럼 동작합니다.

---

## 372줄

```css
color: #ffffff;
```

버튼 글자색을 흰색으로 합니다.

버튼 안에는 `Top`이라는 글자가 있습니다.

```html
<button ...>Top</button>
```

그 글자를 흰색으로 보여줍니다.

---

## 373줄

```css
background: var(--color-primary-strong);
```

버튼 배경색을 강한 primary 색상으로 지정합니다.

즉, 이 버튼은 중요하고 클릭 가능한 버튼처럼 보입니다.

---

## 374줄

```css
border-radius: 50%;
```

버튼을 원형으로 만듭니다.

너비와 높이가 둘 다 52px이고, `border-radius: 50%`이므로 완전한 동그라미가 됩니다.

```txt
정사각형 + border-radius: 50% = 원
```

---

## 375줄

```css
box-shadow: var(--shadow-card);
```

버튼에 그림자를 줍니다.

6강에서 봤던 카드 그림자 변수를 재사용합니다.

버튼이 화면 위에 살짝 떠 있는 느낌을 줍니다.

---

## 376줄

```css
transition: opacity 160ms ease, transform 160ms ease;
```

버튼이 나타나거나 hover될 때 부드럽게 변화하게 합니다.

여기서 부드럽게 바뀌는 속성은 두 개입니다.

```txt
opacity   → 보이기 / 숨기기
transform → hover 시 살짝 위로 이동
```

7강에서 이미 이런 hover 규칙이 있었습니다.

```css
.scroll-top:hover {
  transform: translateY(-2px);
}
```

그래서 hover할 때도 부드럽게 움직입니다.


```txt

```

입니다.

## 378줄

```css
.scroll-top.active {
```

`.scroll-top`이면서 `.active` class도 가진 요소를 선택합니다.

즉, 이런 상태입니다.

```html
<button class="scroll-top active">Top</button>
```

처음 HTML에는 `active`가 없습니다.

JavaScript가 스크롤 위치를 보고, 사용자가 어느 정도 아래로 내려가면 `active`를 붙입니다.

---

## 379줄

```css
opacity: 1;
```

`active` 상태에서는 버튼을 보이게 합니다.

기본값은:

```css

```

이었고, active 상태에서는:

```css

```

이 됩니다.

즉:

```txt
active 없음 → 안 보임
active 있음 → 보임
```

---

## 380줄

```css
pointer-events: auto;
```

`active` 상태에서는 클릭 가능하게 합니다.

기본 상태에서는 클릭을 막았습니다.

```css
pointer-events: none;
```

active 상태에서는 다시 자동으로 돌립니다.

```css
pointer-events: auto;
```

그래서 버튼이 보일 때만 클릭할 수 있습니다.

---

## 381줄

```css
}
```

`.scroll-top.active` 스타일이 끝납니다.

Top 버튼의 핵심은 이것입니다.

```txt
처음:
opacity 0
pointer-events none

스크롤 후 active:
opacity 1
pointer-events auto
```

JavaScript는 버튼의 스타일을 직접 바꾸지 않고, `active` class만 붙였다 뗍니다.

# 10강 핵심 요약

이번 강의는 CSS의 마지막 마무리였습니다.

## 1. Scroll Top 버튼

```css
.scroll-top {
  opacity: 0;
  pointer-events: none;
}

.scroll-top.active {
  opacity: 1;
  pointer-events: auto;
}
```

처음에는 숨겨져 있고 클릭도 안 됩니다.
스크롤 후 JavaScript가 `active`를 붙이면 보이고 클릭 가능해집니다.

---

## 2. Footer

```css
.site-footer {
  display: flex;
  flex-wrap: wrap;
  justify-content: space-between;
}
```

저작권 문구와 소셜 링크를 양쪽으로 배치하고, 좁은 화면에서는 줄바꿈되게 합니다.

---

## 3. Reveal 애니메이션

```css
.reveal {
  opacity: 0;
  transform: translateY(18px);
}

.reveal.visible {
  opacity: 1;
  transform: translateY(0);
}
```

처음에는 살짝 아래에 있고 투명합니다.
화면에 들어오면 JavaScript가 `visible`을 붙이고, CSS가 부드럽게 등장시킵니다.

---

## 4. 모바일 레이아웃

```css
@media (max-width: 767px) {
  ...
}
```

모바일에서는 로고, 다크모드 버튼, 햄버거 버튼의 순서와 위치를 조정합니다.

대략:

```txt
[Shannon Lee]               [Dark] [☰]
```

이런 구조입니다.

---

## 5. 768px 이상 레이아웃

```css
@media (min-width: 768px) {
  ...
}
```

768px 이상에서는 데스크톱형 레이아웃으로 바뀝니다.

```txt
햄버거 버튼 숨김
메뉴 항상 표시
Hero 2열 배치
About / Contact 2열 배치
Skill 3열 배치
Projects 자동 Grid 배치
```

---

## 6. 1024px 이상 레이아웃

```css
@media (min-width: 1024px) {
  .section {
    padding: 5rem 0;
  }
}
```

큰 화면에서는 섹션 여백을 더 넓게 잡고, Hero 설명문도 더 읽기 좋게 조정합니다.

---

# CSS 전체 흐름 최종 정리

이제 `style.css` 전체를 크게 보면 이렇게 됩니다.

```txt
1~68
디자인 변수, 다크모드, 기본 리셋

69~172
헤더, 내비게이션, 햄버거, 다크모드 버튼

173~260
섹션, Hero, 버튼, 필터, 카드 공통 스타일

261~363
Hero 패널, 프로필 카드, 스킬/프로젝트 카드, Contact 폼

364~469
Top 버튼, Footer, Reveal 애니메이션, 반응형 미디어쿼리
```

CSS에서 가장 많이 반복된 사고방식은 이것입니다.

```txt
공통 디자인은 class로 묶는다.
상태 변화는 active, visible, error 같은 class로 표현한다.
화면 크기 변화는 media query로 처리한다.
색상과 간격은 CSS 변수로 관리한다.
```

여기까지가 **10강**, 그리고 **CSS 파트 끝**입니다.

다음 응답을 주시면 **11강: JavaScript 설정값, 상태 객체, DOM 요소 수집**으로 들어가겠습니다.
