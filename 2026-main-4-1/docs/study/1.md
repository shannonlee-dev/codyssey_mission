좋습니다. 이제 HTML을 끝냈고, **CSS 1강**, 전체로는 **6강**으로 들어갑니다.

# 먼저 큰 그림

이 구간의 역할은 세 가지입니다.

```txt
1~18줄   기본 디자인 변수 선언
19~29줄  다크모드용 변수 덮어쓰기
30~42줄  사용자 기기 설정이 다크모드일 때 대응
43~68줄  브라우저 기본 스타일 정리
```

CSS를 처음 볼 때 가장 중요한 관점은 이것입니다.

> CSS는 HTML 요소를 선택한 뒤, 그 요소의 모양을 정한다.

예를 들어:

```css
body {
  background: var(--color-page);
}
```

이 코드는 이렇게 읽습니다.

> `body`를 찾아서 배경색을 `--color-page` 값으로 해라.

---

# 1줄

```css
:root {
```

`:root`는 문서의 최상위 요소를 선택합니다.

HTML 문서에서는 사실상 이 요소를 가리킨다고 보면 됩니다.

```html
<html>
```

즉, `:root`는 전체 웹페이지의 가장 위쪽 기준점입니다.

여기서는 `:root` 안에 CSS 변수를 선언합니다.

CSS 변수는 이렇게 생겼습니다.

```css
--color-page: #f7f4ee;
```

변수는 값을 이름으로 저장해두는 방식입니다.

예를 들어 색상을 매번 직접 쓰는 대신:

```css
background: #f7f4ee;
```

이렇게 변수 이름으로 쓸 수 있습니다.

```css
background: var(--color-page);
```

이렇게 하면 좋은 점이 있습니다.

```txt
색상을 한 곳에서 관리할 수 있다.
다크모드로 바꾸기 쉽다.
전체 디자인의 일관성이 좋아진다.
```

---

# 2줄

```css
--color-page: #f7f4ee;
```

페이지 전체 배경색을 저장한 변수입니다.

변수 이름을 분해해보면:

```txt
--color-page
```

뜻은:

> 페이지 배경에 사용할 색상

입니다.

값은:

```css
#f7f4ee
```

이건 밝고 따뜻한 베이지 계열 색상입니다.

나중에 실제로 이 변수는 `body`에서 사용됩니다.

```css
body {
  background: var(--color-page);
}
```

즉, 2줄은 아직 배경을 칠하는 줄이 아닙니다.

정확히는:

> “페이지 배경색으로 쓸 값을 미리 저장해두는 줄”

입니다.

---

# 3줄

```css
--color-surface: #ffffff;
```

`surface`는 카드, 패널, 폼 같은 표면 색상을 의미합니다.

예를 들어 이런 요소들이 있습니다.

```txt
Hero 패널
프로필 카드
스킬 카드
프로젝트 카드
Contact 폼
```

이런 박스들의 배경에 `--color-surface`가 쓰입니다.

값은 흰색입니다.

```css
#ffffff
```

즉, 밝은 테마에서는:

```txt
전체 페이지 배경 → 살짝 베이지색
카드 표면 → 흰색
```

이렇게 구분됩니다.

---

# 4줄

```css
--color-text: #24201d;
```

기본 글자색입니다.

`text`니까 본문 글자, 제목 글자 등에 쓰입니다.

값은 완전한 검정색 `#000000`이 아니라 약간 따뜻한 어두운 색입니다.

왜 완전 검정을 안 쓸까요?

완전 검정은 밝은 배경에서 너무 강하게 느껴질 수 있습니다.
그래서 살짝 부드러운 어두운 색을 자주 씁니다.

나중에 `body`에서 이렇게 사용됩니다.

```css
color: var(--color-text);
```

---

# 5줄

```css
--color-muted: #6d6259;
```

`muted`는 약한 글자색입니다.

중요도가 낮은 설명문, 보조 텍스트에 사용됩니다.

예를 들어 HTML에서 이런 문장들이 있었죠.

```html
<p class="hero-copy">
```

또는 프로젝트 카드 설명문 같은 곳입니다.

주요 제목은 진한 `--color-text`를 쓰고, 설명문은 `--color-muted`를 쓰면 정보의 위계가 생깁니다.

```txt
중요한 글자 → 진하게
보조 설명 → 살짝 연하게
```

---

# 6줄

```css
--color-primary: #0f766e;
```

대표 색상입니다.

`primary`는 사이트의 핵심 브랜드 색상처럼 쓰입니다.

버튼, 강조 배경, 링크, 포커스 효과 등에 사용할 수 있습니다.

이 프로젝트에서는 청록색 계열입니다.

`primary`는 보통 이런 곳에 쓰입니다.

```txt
강조 버튼
활성 상태
포커스 테두리
카드 hover 효과
```

---

# 7줄

```css
--color-primary-strong: #115e59;
```

`primary`보다 더 진한 대표 색상입니다.

이름에 `strong`이 붙었죠.

```txt
primary        → 기본 강조색
primary-strong → 더 강한 강조색
```

예를 들어 primary 버튼의 배경색으로 사용됩니다.

```css
.button.primary {
  background: var(--color-primary-strong);
}
```

더 진한 색을 버튼 배경으로 쓰면 흰 글자와 대비가 잘 납니다.

---

# 8줄

```css
--color-accent: #d94655;
```

보조 강조 색상입니다.

`accent`는 primary와는 다른 느낌으로 주의를 끌 때 사용합니다.

이 프로젝트에서는 붉은 계열 색상입니다.

예를 들어:

```txt
eyebrow 라벨
에러 메시지
상태 오류 표시
```

같은 곳에 쓰입니다.

즉, `primary`가 사이트의 기본 강조라면, `accent`는 특별히 눈에 띄게 할 때 쓰는 색상입니다.

---

# 9줄

```css
--color-border: #ded6ca;
```

테두리 색상입니다.

카드, 버튼, 입력창에 테두리가 필요할 때 이 변수를 씁니다.

예를 들어 나중에 이런 코드가 나옵니다.

```css
border: 1px solid var(--color-border);
```

이렇게 해두면 모든 테두리 색상을 한 변수로 통일할 수 있습니다.

---

# 10줄

```css
--font-base: system-ui, -apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif;
```

기본 글꼴 묶음입니다.

이 줄은 조금 길지만, 의미는 간단합니다.

> 사용자의 운영체제에서 가장 자연스러운 기본 UI 글꼴을 사용하자.

하나씩 보면:

```txt
system-ui
-apple-system
BlinkMacSystemFont
"Segoe UI"
sans-serif
```

이건 우선순위 목록입니다.

브라우저는 앞에서부터 가능한 글꼴을 찾습니다.

예를 들어 Mac이면 Apple 시스템 글꼴을 쓰고, Windows면 Segoe UI를 쓸 수 있습니다.

마지막의 `sans-serif`는 안전장치입니다.

앞의 글꼴들이 없으면 기본 고딕 계열 글꼴을 쓰라는 뜻입니다.

나중에 `body`에서 이렇게 적용합니다.

```css
font-family: var(--font-base);
```

---

# 11줄

```css
--space-1: 0.5rem;
```

간격 변수입니다.

`space`는 여백을 의미합니다.

`0.5rem`은 보통 기본 글자 크기의 절반 정도입니다.

대부분의 브라우저에서 기본 글자 크기가 16px이므로:

```txt
1rem ≈ 16px
0.5rem ≈ 8px
```

정도로 생각하면 됩니다.

이 변수는 작은 간격에 사용됩니다.

```txt
작은 버튼 간격
작은 제목 아래 여백
작은 카드 내부 간격
```

---

# 12줄

```css
--space-2: 1rem;
```

두 번째 간격 변수입니다.

```txt
1rem ≈ 16px
```

가장 기본적인 여백 단위로 자주 쓰입니다.

예를 들어 메뉴 간격, 카드 사이 간격 등에 사용됩니다.

---

# 13줄

```css
--space-3: 1.5rem;
```

조금 더 큰 간격입니다.

```txt
1.5rem ≈ 24px
```

카드 안쪽 패딩이나 섹션 내부 요소 간격에 쓰기 좋습니다.

---

# 14줄

```css
--space-4: 2rem;
```

더 큰 간격입니다.

```txt
2rem ≈ 32px
```

Hero 영역의 큰 요소 간격이나 섹션 내부의 넓은 간격에 사용할 수 있습니다.

---

# 15줄

```css
--space-5: 3rem;
```

가장 큰 간격 변수입니다.

```txt
3rem ≈ 48px
```

섹션 위아래 패딩처럼 큰 여백에 사용됩니다.

나중에 이런 코드가 나옵니다.

```css
.section {
  padding: var(--space-5) 0;
}
```

즉, 각 섹션의 위아래에 넉넉한 공간을 줍니다.

---

# 16줄

```css
--shadow-card: 0 14px 34px rgba(36, 32, 29, 0.12);
```

카드 그림자 변수입니다.

값을 나눠보면:

```txt
0       → 가로 방향 그림자 이동 없음
14px    → 아래쪽으로 14px 이동
34px    → 34px 정도 흐림
rgba(...) → 그림자 색상
```

`rgba(36, 32, 29, 0.12)`에서 마지막 `0.12`는 투명도입니다.

```txt
0   → 완전 투명
1   → 완전 불투명
0.12 → 아주 은은함
```

즉, 이 그림자는 강한 그림자가 아니라 부드러운 카드 그림자입니다.

나중에 카드류 요소에서 이렇게 사용됩니다.

```css
box-shadow: var(--shadow-card);
```

---

# 17줄

```css
--header-height: 72px;
```

헤더 높이를 저장한 변수입니다.

상단 헤더는 페이지 위에 고정되어 있습니다.

이 높이 값은 여러 곳에서 재사용됩니다.

예를 들어 Hero 영역 높이를 계산할 때:

```css
min-height: calc(100vh - var(--header-height));
```

이런 식으로 씁니다.

뜻은:

```txt
화면 전체 높이에서 헤더 높이만큼 빼라
```

또 모바일 메뉴 위치를 잡을 때도 사용됩니다.

```css
inset: var(--header-height) 1rem auto 1rem;
```

즉, 헤더 아래에서 메뉴가 열리도록 만들 수 있습니다.

---

```css
}
```

`:root` 블록이 끝났습니다.

1~18줄 전체는 밝은 테마의 기본 디자인 토큰입니다.

```txt

글꼴
간격
그림자
헤더 높이
```

를 한 곳에 모아둔 것입니다.

# 19줄

```css
[data-theme="dark"] {
```

여기서 다크모드 설정이 시작됩니다.

이 선택자는 이렇게 읽습니다.

> `data-theme="dark"` 속성을 가진 요소를 찾아라.

1강에서 HTML 시작 줄이 이랬습니다.

```html
<html lang="ko" data-theme="light">
```

JavaScript가 다크모드 버튼을 누르면 이 값이 바뀔 수 있습니다.

```html
<html lang="ko" data-theme="dark">
```

그러면 CSS의 이 부분이 적용됩니다.

즉:

```txt
data-theme="light" → :root의 기본 밝은 변수 사용
data-theme="dark"  → 19~29줄의 어두운 변수로 덮어쓰기
```

# 20줄

```css
--color-page: #141819;
```

다크모드에서 페이지 전체 배경색을 바꿉니다.

```css

```

다크 테마에서는 어두운 색으로 바뀝니다.

```css
--color-page: #141819;
```

# 21줄

다크모드에서 카드, 폼, 패널의 표면 색상입니다.

```css
#ffffff
```

다크 테마에서는 페이지 배경보다 살짝 밝은 어두운 색을 사용합니다.

```css
#202628
```

```css
#24201d
```

다크모드에서는 배경이 어두우므로 글자가 밝아져야 합니다.

```css
#f3efe8
```

그래야 읽을 수 있습니다.

```css
#6d6259
```

그래서 밝은 회색/베이지 계열로 바꿉니다.

다크모드의 기본 강조색입니다.

어두운 배경에서는 너무 진한 색보다 밝고 선명한 색이 잘 보입니다.

```css
--color-primary-strong: #2dd4bf;
```

다크모드의 강한 강조색입니다.

밝은 테마에서는 `#115e59`처럼 어두운 청록을 썼지만, 다크모드에서는 더 밝은 청록을 사용합니다.

# 26줄

```css
--color-accent: #fb7185;
```

다크모드의 보조 강조색입니다.

# 31줄

```css
:root:not([data-theme="light"]) {
```

이 줄은 조금 어렵지만 아주 재미있습니다.

쪼개서 봅시다.

```css
:root
```

문서 최상위 요소를 뜻합니다.

```css
:not(...)
```

괄호 안 조건이 아닌 경우를 뜻합니다.

```css
[data-theme="light"]
```

`data-theme` 값이 `light`인 요소를 뜻합니다.

전체를 합치면:

```css
:root:not([data-theme="light"])
```

뜻은:

> 최상위 요소인데, `data-theme="light"`가 아닌 경우

입니다.

왜 이렇게 했을까요?

사용자가 직접 라이트모드를 선택한 경우에는 운영체제가 다크모드여도 라이트모드를 유지해야 하기 때문입니다.

즉:

```txt
사용자가 직접 light 선택
→ light를 존중

사용자가 직접 light라고 정하지 않음
+ 기기 설정이 dark
→ dark 변수 적용
```

이 프로젝트에서는 HTML이 처음부터 `data-theme="light"`를 가지고 있긴 합니다.
그래서 실제 동작에서는 JavaScript가 테마 값을 어떻게 관리하는지가 중요해집니다.

# 44줄

```css
box-sizing: border-box;
```

이 줄은 CSS 레이아웃에서 아주 중요합니다.

기본적으로 CSS에서 요소의 너비를 계산할 때 헷갈릴 수 있습니다.

예를 들어 어떤 박스에 이렇게 줬다고 합시다.

```css
width: 100px;
padding: 20px;
border: 1px solid black;
```

기본 계산 방식에서는 실제 차지하는 너비가 100px보다 커질 수 있습니다.

하지만:

```css
box-sizing: border-box;
```

를 쓰면 `width` 안에 padding과 border까지 포함해서 계산합니다.

즉:

```txt
border-box 사용 전:
  콘텐츠 너비 + padding + border = 실제 너비

border-box 사용 후:
  지정한 width 안에 콘텐츠 + padding + border 포함
```

그래서 레이아웃 계산이 훨씬 직관적입니다.

이 프로젝트에서는 모든 요소에 적용합니다.

# 이번 강의의 핵심

이번 1~68줄은 “화려한 디자인”이 아니라 **디자인 시스템의 기초 공사**입니다.

## 1. CSS 변수

```css
:root {
  --color-page: #f7f4ee;
}
```

변수를 만들어두고:

```css
background: var(--color-page);
```

이렇게 재사용합니다.

---

## 2. 다크모드 구조

```css
[data-theme="dark"] {
  --color-page: #141819;
}
```

HTML의 이 값이:

```html
<html data-theme="dark">
```

이렇게 바뀌면 CSS 변수 값도 바뀝니다.

그래서 전체 화면 색이 바뀝니다.

---

## 3. 기본 리셋

```css
* {
  box-sizing: border-box;
}

body {
  margin: 0;
}

a {
  text-decoration: none;
}

img {
  max-width: 100%;
}

button,
input,
textarea {
  font: inherit;
}
```

브라우저마다 기본 스타일이 조금씩 다를 수 있기 때문에, 프로젝트가 원하는 기준으로 기본값을 정리합니다.

```txt
→ 그 변수를 사용해서 실제 배경색, 글자색, 글꼴 적용

나머지 기본 선택자
→ 브라우저 기본 스타일을 정리해서 예측 가능한 디자인 시작
```

특히 이 구조는 정말 중요합니다.

```css
body {
  background: var(--color-page);
  color: var(--color-text);
}
```

body는 변수만 바라봅니다.

```css
:root {
  --color-page: 밝은 배경;
  --color-text: 어두운 글자;
}

[data-theme="dark"] {
  --color-page: 어두운 배경;
  --color-text: 밝은 글자;
}
```

변수 값이 바뀌면 화면도 바뀝니다.

이게 이 프로젝트의 다크모드 핵심 원리입니다.
