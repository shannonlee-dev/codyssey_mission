## 174줄

```css
width: min(1120px, calc(100% - 2rem));
```

섹션의 너비를 정합니다.

7강에서 navbar에도 비슷한 코드가 있었죠.

```css
width: min(1120px, calc(100% - 2rem));
```

이 말은:

```txt
섹션 너비는 최대 1120px까지만 커진다.

```

입니다.

큰 화면에서 본문이 너무 넓게 퍼지면 읽기 어렵습니다.

```txt
너무 넓은 본문:
한 줄이 너무 길어져서 시선 이동이 힘듦
```

그래서 최대 너비를 `1120px`로 제한합니다.

하지만 모바일에서는 1120px이 화면보다 훨씬 크죠.
그럴 때는 `calc(100% - 2rem)`이 선택됩니다.

```txt
작은 화면:
화면 너비에서 좌우 합쳐 2rem 정도 여백을 남김
```

즉, 이 한 줄이 데스크톱과 모바일을 동시에 고려합니다.

`.hero` class를 가진 요소를 선택합니다.

HTML에서는 첫 화면 섹션이었습니다.

```html
<section class="hero section reveal" id="hero">
```

Hero는 사용자가 처음 보는 가장 중요한 화면이므로 별도 스타일이 필요합니다.

## 179줄

```css
min-height: calc(100vh - var(--header-height));
```

Hero 영역의 최소 높이를 정합니다.

여기에는 두 가지 중요한 단위와 함수가 나옵니다.

```css
100vh
```

`vh`는 viewport height입니다.

```txt
100vh = 현재 브라우저 화면 높이 100%
```

즉, 화면 전체 높이를 의미합니다.

그런데 이 사이트에는 위에 헤더가 있습니다.

6강에서 헤더 높이 변수를 봤죠.

```css
--header-height: 72px;
```

그래서 Hero 높이를 이렇게 계산합니다.

```css
calc(100vh - var(--header-height))
```

뜻은:

```txt
브라우저 화면 전체 높이에서 헤더 높이를 뺀 만큼
```

입니다.

즉, Hero 영역이 헤더 아래 남은 화면을 거의 꽉 채우게 됩니다.

```txt
┌───────────────┐
│ Header 72px   │
├───────────────┤
│               │
│ Hero          │  ← 남은 화면 높이만큼
│               │
└───────────────┘
```

## 180줄

```css
display: grid;
```

Hero 섹션을 CSS Grid 컨테이너로 만듭니다.

HTML에서 Hero 안에는 두 큰 덩어리가 있었습니다.

```html
<div class="hero-content">...</div>
<div class="hero-panel">...</div>
```

즉:

```txt
hero-content → 왼쪽 또는 위쪽 텍스트 영역
hero-panel   → 오른쪽 또는 아래쪽 상태 흐름 패널
```

`display: grid`를 쓰면 이 두 덩어리를 배치하기 쉬워집니다.

현재 기본 상태에서는 자세한 열 설정은 아직 없습니다.
나중에 미디어쿼리에서 화면이 넓을 때 두 열로 바꾸는 코드가 나옵니다.

지금은 “Hero 안의 요소들을 Grid 레이아웃으로 관리하겠다” 정도로 보면 됩니다.

```css
align-items: center;
```

```txt

```

`align-items: center`를 주면 Hero 내부 콘텐츠가 세로 중앙에 더 가깝게 배치됩니다.

그래서 최대 너비를 `780px`로 제한합니다.

## 192줄

```css
font-size: clamp(2.6rem, 12vw, 4.8rem);
```

Hero 제목의 글자 크기를 정합니다.

여기서 `clamp()`가 중요합니다.

`clamp()`는 값을 최소값과 최대값 사이로 제한합니다.

구조는 이렇습니다.

```css
clamp(최소값, 선호값, 최대값)
```

여기서는:

```txt
최소값: 2.6rem
선호값: 12vw
최대값: 4.8rem
```

입니다.

즉:

```txt
화면이 작아도 h1은 최소 2.6rem 이상
화면 크기에 따라 12vw 기준으로 유동적으로 커짐
하지만 아무리 커져도 최대 4.8rem까지만
```

입니다.

`vw`는 viewport width입니다.

```txt
12vw = 화면 너비의 12%
```

그래서 화면이 넓어지면 제목도 어느 정도 커집니다.

하지만 `4.8rem`을 넘지 않도록 막습니다.

이 한 줄 덕분에 Hero 제목은 반응형으로 자연스럽게 커지고 작아집니다.

## 260줄

# 이번 강의의 핵심 흐름

이번 구간은 “반복되는 디자인 규칙을 공통 class로 묶는 방식”이 핵심입니다.

## 1. 모든 섹션은 같은 폭과 여백을 공유한다

```css
.section {
  width: min(1120px, calc(100% - 2rem));
  margin: 0 auto;
  padding: var(--space-5) 0;
}
```

이 덕분에 About, Skills, Projects, Contact가 같은 레이아웃 규칙을 가집니다.

---

## 2. Hero는 화면 첫인상에 맞게 크게 잡는다

```css
.hero {
  min-height: calc(100vh - var(--header-height));
  display: grid;
  align-items: center;
}
```

Hero는 헤더 아래 화면을 크게 차지하고, 내부 콘텐츠를 중앙에 배치합니다.

---

## 3. 버튼은 공통 스타일 + 변형 스타일로 나뉜다

공통 버튼:

```css
.button {
  display: inline-flex;
  min-height: 44px;
  ...
}
```

Primary 버튼:

```css
.button.primary {
  color: #ffffff;
  background: var(--color-primary-strong);
}
```

Secondary 버튼:

```css
.button.secondary {
  color: var(--color-text);
  background: var(--color-surface);
  border: 1px solid var(--color-border);
}
```

즉:

```txt
button       → 모든 버튼 공통 뼈대
primary      → 중요한 버튼
secondary    → 보조 버튼
small        → 작은 버튼
```

이런 구조입니다.

---

## 4. 필터 버튼은 JavaScript가 만들지만 CSS는 미리 준비한다

HTML에는 이 자리만 있었습니다.

```html
<div class="filter-bar" data-filter-bar></div>
```

하지만 CSS에는 이미 `.filter-button` 스타일이 있습니다.

```css
.filter-button {
  ...
}

.filter-button.active {
  ...
}
```

즉:

```txt
HTML은 필터 버튼이 들어갈 자리만 준비
CSS는 필터 버튼의 디자인을 미리 준비
JavaScript는 실제 버튼을 생성하고 active를 바꿈
```

입니다.

---

## 5. 카드류 요소는 공통 박스 스타일을 공유한다

```css
.hero-panel,
.profile-card,
.skill-card,
.project-card,
.contact-form {
  background: var(--color-surface);
  border: 1px solid var(--color-border);
  border-radius: 8px;
  box-shadow: var(--shadow-card);
}
```

이 덕분에 서로 다른 요소들이 같은 디자인 언어를 가집니다.

```txt
Hero 패널
프로필 카드
스킬 카드
프로젝트 카드
Contact 폼
```

전부 같은 표면색, 테두리, 둥근 모서리, 그림자를 사용합니다.

---

# 8강 최종 요약

이번 강의는 “본문 UI의 공통 규칙”을 만드는 구간이었습니다.

```txt
.section
→ 본문 구역의 폭과 여백

.hero
→ 첫 화면의 높이와 배치

h1, h2
→ 큰 제목 스타일

.eyebrow
→ 작은 섹션 라벨

.hero-actions / section-header / filter-bar / social-links
→ 여러 요소를 flex로 나열하는 그룹

.button
→ 버튼 공통 스타일

.button.primary / .button.secondary / .button.small
→ 버튼 변형 스타일

.filter-button / .filter-button.active
→ 프로젝트 필터 버튼 상태 스타일

card류 요소들
→ 패널, 카드, 폼의 공통 박스 디자인
```

여기서 가장 중요한 사고방식은 이것입니다.

```txt
반복되는 디자인은 공통 class로 묶는다.
특별한 차이는 추가 class로 표현한다.
상태 변화는 active 같은 class로 표현한다.
```

예를 들면:

```html
<a class="button primary">View Projects</a>
<a class="button secondary">Contact</a>
<button class="button secondary small">다시 시도</button>
<button class="filter-button active">All</button>
```

이런 식입니다.

`button`은 공통 뼈대,
`primary`, `secondary`, `small`, `active`는 상태나 변형입니다.
