## 70줄

```css
position: sticky;
```

이 줄은 헤더가 스크롤할 때 어떻게 움직일지를 정합니다.

`sticky`는 평소에는 일반 요소처럼 있다가, 특정 위치에 도달하면 화면에 붙는 방식입니다.

여기서는 헤더를 화면 위쪽에 붙게 만들 겁니다.

느낌은 이렇습니다.

```txt
페이지를 아래로 스크롤해도
상단 메뉴는 화면 위에 계속 붙어 있음
```

`position: fixed`와 비슷해 보이지만 조금 다릅니다.

```txt
fixed  → 처음부터 무조건 화면에 고정
sticky → 원래 위치에 있다가 조건에 맞으면 붙음
```

이 프로젝트에서는 헤더가 맨 위에 있으므로 실제로는 거의 고정 헤더처럼 느껴집니다.

## 71줄

`sticky`가 붙을 위치를 정합니다.

즉, 헤더가 화면 맨 위에 붙습니다.

70~71줄은 같이 봐야 합니다.

```css

```

이 둘이 합쳐져서:


가 됩니다.

## 72줄

```css
z-index: 20;
```

`z-index`는 요소의 앞뒤 쌓임 순서를 정합니다.

웹페이지는 평면처럼 보이지만, 실제로는 요소들이 겹칠 수 있습니다.

```txt
z-index가 큰 요소 → 더 앞에 보임
z-index가 작은 요소 → 뒤에 깔림
```

헤더는 페이지 위에 항상 보여야 합니다.

예를 들어 Hero, 카드, 이미지가 스크롤되더라도 헤더가 그 밑에 묻히면 안 됩니다.

그래서:

```css
z-index: 20;
```

을 줘서 헤더를 앞쪽 레이어에 둡니다.

## 73줄

```css
background: color-mix(in srgb, var(--color-page) 88%, transparent);
```

헤더의 배경색을 정합니다.

조금 어려워 보이지만 나눠서 보면 됩니다.

```css
var(--color-page)
```

6강에서 본 페이지 배경색 변수입니다.

밝은 테마에서는 대략 베이지색이고, 다크모드에서는 어두운 색으로 바뀝니다.

```css
transparent
```

는 투명입니다.

```css
color-mix(in srgb, var(--color-page) 88%, transparent)
```

뜻은:

> `--color-page` 색을 88% 정도 쓰고, 나머지는 투명하게 섞어라.

즉, 헤더 배경을 완전히 불투명하게 하지 않고 살짝 투명한 느낌으로 만듭니다.

이렇게 하면 뒤쪽 콘텐츠가 아주 은은하게 비치는 듯한 효과를 줄 수 있습니다.

## 74줄

```css
border-bottom: 1px solid transparent;
```

헤더 아래쪽 테두리를 만듭니다.

```css
1px
```

두께는 1px입니다.

```css
solid
```

실선입니다.

```css
transparent
```

색상은 투명입니다.

그런데 이상하죠.

> 투명한 테두리를 왜 만들까?

이유는 나중에 스크롤 상태에서 테두리 색만 바꾸기 위해서입니다.

처음에는 테두리가 안 보입니다.

```css
border-bottom: 1px solid transparent;
```

스크롤하면 나중에 이 코드가 적용됩니다.

```css
border-color: var(--color-border);
```

즉, 테두리 공간은 처음부터 잡아두고, 색만 투명에서 실제 색으로 바꿉니다.

이렇게 하면 스크롤할 때 레이아웃이 흔들리지 않습니다.

## 75줄

```css
backdrop-filter: blur(16px);
```

이 줄은 헤더 뒤쪽 배경을 흐리게 만듭니다.

`backdrop-filter`는 요소 자체가 아니라 **요소 뒤에 보이는 것**에 필터를 적용합니다.

```css
blur(16px)
```

는 16px 정도 흐림 효과입니다.

즉, 헤더가 살짝 투명하고, 뒤에 있는 내용은 흐릿하게 보입니다.

유리판 같은 느낌을 만드는 데 자주 쓰입니다.

```txt
투명한 헤더
+ 뒤쪽 blur
= glassmorphism 느낌
```

## 76줄

```css
transition: background-color 180ms ease, border-color 180ms ease, box-shadow 180ms ease;
```

헤더의 변화가 부드럽게 일어나도록 합니다.

`transition`은 CSS 값이 바뀔 때 애니메이션처럼 천천히 바뀌게 하는 속성입니다.

여기서는 세 가지 변화를 부드럽게 만듭니다.

```txt
background-color
border-color
box-shadow
```

각각:

```txt

ease  → 자연스럽게 시작하고 끝나는 속도감
```

입니다.

나중에 JavaScript가 스크롤 상태에 따라 `.scrolled` 클래스를 붙이면 헤더 배경, 테두리, 그림자가 바뀝니다.

그 변화가 갑자기 딱 바뀌지 않고 부드럽게 바뀌게 됩니다.



## 85줄

```css
width: min(1120px, calc(100% - 2rem));
```

navbar의 너비를 정합니다.

이 줄은 반응형 레이아웃에서 중요합니다.

먼저:

```css
min(A, B)
```

는 A와 B 중 더 작은 값을 선택합니다.

여기서는 두 값이 있습니다.

```css
1120px
```

그리고:

```css
calc(100% - 2rem)
```

`calc`는 계산 함수입니다.

화면 전체 너비에서 `2rem`을 뺍니다.

즉, 좌우에 최소한의 여백을 남깁니다.

이 줄을 해석하면:

```txt
navbar 너비는 최대 1120px까지만 커진다.
하지만 화면이 좁으면 화면 너비에서 2rem을 뺀 만큼만 사용한다.
```

예를 들어:

```txt
큰 화면:
  min(1120px, 1400px - 2rem)
  → 1120px

작은 화면:
  min(1120px, 390px - 2rem)
  → 390px - 2rem
```

그래서 큰 화면에서는 너무 넓어지지 않고, 작은 화면에서는 화면 밖으로 튀어나가지 않습니다.

# 모바일 메뉴 기본 스타일

## 96줄


HTML에서는 이 부분입니다.


상단 메뉴 목록입니다.

```txt

```

---

## 97줄

```css
position: fixed;
```

메뉴 목록을 화면 기준으로 고정 배치합니다.

`fixed`는 viewport, 즉 브라우저 화면을 기준으로 위치를 잡습니다.

이건 모바일 메뉴에서 중요합니다.

햄버거 버튼을 눌렀을 때 메뉴가 헤더 아래에 떠 있는 패널처럼 나타나야 하기 때문입니다.

---

## 98줄

```css
inset: var(--header-height) 1rem auto 1rem;
```

`inset`은 `top`, `right`, `bottom`, `left`를 한 번에 지정하는 속성입니다.

순서는 이렇게 읽습니다.

```txt
top right bottom left
```

따라서 이 줄은:

```css
top: var(--header-height);
right: 1rem;
bottom: auto;
left: 1rem;
```

와 비슷합니다.

`var(--header-height)`는 72px입니다.

즉, 메뉴는 헤더 높이만큼 아래에서 시작합니다.

```txt
화면 맨 위
┌──────────── 헤더 72px ────────────┐
└──────────── 메뉴는 여기 아래부터 ──┘
```

좌우는 `1rem`씩 띄웁니다.

그래서 모바일 메뉴가 화면 좌우에 딱 붙지 않고 여백을 가집니다.

---

## 99줄

```css
display: none;
```

기본적으로 메뉴를 숨깁니다.

모바일에서는 처음부터 메뉴 목록을 다 보여주지 않고, 햄버거 버튼을 눌렀을 때만 보여줍니다.

```txt
처음 상태:
메뉴 안 보임

햄버거 클릭:
메뉴 보임
```

이 줄이 “처음에는 안 보이게” 만드는 역할입니다.

---

`display: none` 상태에서는 당장 보이지 않지만, 나중에 `.active`가 붙으면 `display: flex`가 됩니다.

그때 이 규칙이 적용됩니다.

# Hamburger 버튼 박스

## 132줄

```css
.hamburger {
```

`.hamburger` class를 가진 버튼을 선택합니다.

HTML에서는 이 부분입니다.

```html
<button class="hamburger" type="button" aria-label="메뉴 열기" ...>
  <span></span>
  <span></span>
  <span></span>
</button>
```

이 버튼 안의 `span` 세 개가 햄버거 선 3개가 됩니다.

---

## 133줄

```css
display: inline-flex;
```

햄버거 버튼을 inline-flex 컨테이너로 만듭니다.

`inline-flex`는 바깥으로는 inline처럼 흐르지만, 안쪽 자식들은 flex로 배치할 수 있습니다.

여기서 중요한 건 안쪽 `span` 3개를 정렬하려고 flex를 쓴다는 점입니다.

---

## 134줄

```css
width: 42px;
```

햄버거 버튼의 너비입니다.

42px짜리 정사각형 버튼을 만들 준비입니다.

---

## 135줄

```css
height: 42px;
```

햄버거 버튼의 높이입니다.

너비도 42px, 높이도 42px이므로 정사각형입니다.

```txt
42px × 42px 버튼
```

모바일에서 손가락으로 누르기 적당한 크기입니다.

---

## 136줄

```css
align-items: center;
```

버튼 안의 `span`들을 가로 방향 가운데에 정렬합니다.

현재 flex 방향은 뒤에서 column으로 바뀝니다.

column일 때 `align-items`는 가로축 정렬입니다.

즉, 선 3개가 버튼의 가로 중앙에 놓입니다.

---

## 137줄

```css
justify-content: center;
```

버튼 안의 `span`들을 세로 방향 가운데에 정렬합니다.

즉, 선 3개 묶음이 버튼 중앙에 옵니다.

136~137줄을 합치면:

```txt
span 3개를 버튼 한가운데에 배치
```

입니다.

---

## 138줄

```css
flex-direction: column;
```

# Hover 효과 공통 처리

## 168~170줄

```css

```

여러 요소의 hover 상태를 한 번에 선택합니다.

```txt
theme-toggle:hover → 다크모드 버튼에 마우스를 올렸을 때
```

```css
.button:hover
```

즉, 이런 요소들이 대상입니다.

```html
<a class="button primary" href="#projects">View Projects</a>
<button class="button primary" type="submit">Send</button>
```

# 이번 강의의 핵심 흐름

이번 구간은 상단 인터페이스의 동작 준비가 핵심입니다.

## 1. 헤더는 화면 위에 붙는다

```css
.site-header {
  position: sticky;
  top: 0;
  z-index: 20;
}
```

스크롤해도 상단 메뉴가 계속 보입니다.

---

## 2. 스크롤하면 헤더 스타일이 바뀐다

```css
.site-header.scrolled {
  background: var(--color-surface);
  border-color: var(--color-border);
  box-shadow: ...
}
```

`scrolled` class는 JavaScript가 붙입니다.

```txt
스크롤 전 → 투명한 느낌
스크롤 후 → 배경, 테두리, 그림자 생김
```

---

## 3. navbar는 flex로 정렬한다

```css
.navbar {
  display: flex;
  align-items: center;
  justify-content: space-between;
}
```

로고, 메뉴, 버튼들이 한 줄 안에서 정렬됩니다.

---

## 4. 모바일 메뉴는 기본적으로 숨겨져 있다

```css
.nav-menu {
  display: none;
}
```

처음에는 안 보입니다.

---

## 5. active가 붙으면 메뉴가 열린다

```css
.nav-menu.active {
  display: flex;
}
```

JavaScript는 메뉴를 직접 그리는 게 아니라, class만 붙였다 뗍니다.

```txt
active 없음 → 메뉴 닫힘
active 있음 → 메뉴 열림
```

---

## 6. 햄버거 버튼도 active로 X 모양이 된다

```css
.hamburger.active span:nth-child(1) {
  transform: translateY(7px) rotate(45deg);
}

.hamburger.active span:nth-child(2) {
  opacity: 0;
}

.hamburger.active span:nth-child(3) {
  transform: translateY(-7px) rotate(-45deg);
}
```

즉:

```txt
햄버거 닫힘:
────
────
────

햄버거 열림:
╲  ╱
 ╳
╱  ╲
```

# 7강 최종 요약

이번 강의에서 가장 중요한 개념은 **class 상태 전환**입니다.

HTML은 처음에 이렇게 있습니다.

```html
<ul class="nav-menu" id="nav-menu">
<button class="hamburger">
<header class="site-header">
```

JavaScript가 상황에 따라 class를 추가합니다.

```html
<ul class="nav-menu active" id="nav-menu">
<button class="hamburger active">
<header class="site-header scrolled">
```

CSS는 그 class 상태에 맞춰 화면을 바꿉니다.

```txt
scrolled → 헤더가 더 또렷해짐
active nav-menu → 모바일 메뉴 열림
active hamburger → 햄버거가 X로 바뀜
```

즉, 이 프로젝트의 중요한 패턴은 이것입니다.

```txt
JavaScript가 상태 class를 붙인다.
CSS가 그 class를 보고 화면을 바꾼다.
```

여기까지가 **7강**입니다.
다음 응답을 주시면 **8강: Section, Hero, Button, Card 공통 스타일**로 이어가겠습니다.
