## 353줄

```css
min-height: 1.25rem;
```

에러 메시지 영역의 최소 높이입니다.

이게 아주 좋은 포인트입니다.

왜 비어 있는 에러 메시지에도 높이를 줄까요?

에러가 없을 때는 빈칸이고, 에러가 생기면 텍스트가 들어옵니다.

만약 최소 높이가 없으면, 에러 메시지가 생기는 순간 폼 전체가 아래로 밀릴 수 있습니다.

```txt
에러 없음 → 공간 없음
에러 발생 → 갑자기 공간 생김 → 레이아웃 흔들림
```

`min-height`를 주면 에러 메시지가 없어도 자리를 조금 확보합니다.

```txt
에러 없음 → 빈 공간 유지
에러 발생 → 그 공간에 메시지 표시
```

그래서 화면이 덜 흔들립니다.

# 이번 강의 핵심 요약

이번 구간은 크게 네 가지입니다.

## 1. Hero 패널 내부 정리

```css
.hero-panel {
  padding: var(--space-3);
}

.hero-panel ol {
  display: grid;
  gap: var(--space-1);
}
```

스크린샷 오른쪽의 `STATE FLOW` 카드 안쪽 여백과 목록 간격을 만듭니다.

---

## 2. About / Contact는 grid 구조

```css
.about,
.contact {
  display: grid;
  gap: var(--space-3);
}
```

About은:

```txt
소개 텍스트 + 프로필 카드
```

Contact는:

```txt
설명 텍스트 + 폼
```

구조로 배치됩니다.

---

## 3. Skill / Project 카드는 hover 효과가 있다

```css
.skill-card:hover,
.project-card:hover {
  transform: translateY(-4px);
  border-color: var(--color-primary);
  box-shadow: 0 18px 40px rgba(15, 118, 110, 0.18);
}
```

마우스를 올리면 카드가 살짝 떠오릅니다.

여기서도 중요한 패턴은:

```txt
기본 상태
hover 상태
```

를 CSS가 나눠 관리한다는 것입니다.

---

## 4. Contact Form은 사용성과 접근성을 고려한다

```css
.form-field input:focus,
.form-field textarea:focus {
  outline: none;
  border-color: var(--color-primary);
  box-shadow: 0 0 0 3px color-mix(...);
}
```

기본 outline은 없애지만, 대신 직접 focus 표시를 만듭니다.

그리고 에러 메시지에는 `min-height`를 줘서 레이아웃 흔들림을 줄입니다.

```css
.field-error {
  min-height: 1.25rem;
}
```

# 9강 최종 정리

이번 강의의 핵심 사고방식은 이것입니다.

```txt
카드 안쪽에는 padding을 준다.
카드 사이에는 grid gap을 준다.
카드는 hover 상태에서 살짝 반응하게 한다.
입력창은 focus 상태를 분명하게 보여준다.
에러 메시지 자리는 미리 높이를 확보해서 화면 흔들림을 줄인다.
```

지금까지 CSS에서 계속 반복되는 패턴도 보입니다.

```txt
공통 스타일
→ hero-panel, profile-card, skill-card, project-card, contact-form

상태 스타일
→ :hover, :focus, .error

반응 준비
→ grid, flex-wrap, min-height
```

여기까지가 **9강**입니다.
